# Nmap 参数策略方案

## 文档定位

- **用途**：针对几千个公网 IP 进行一次性 HTTP 端口扫描的 Nmap 参数策略说明
- **适用场景**：一次性任务、仅关注 HTTP/HTTPS 常见端口、分钟级耗时要求、无后续系统对接
- **后续衔接**：扫描结果将用于批量访问特定 URI

---

## 扫描策略总览

| 项目 | 策略 |
|------|------|
| 扫描类型 | SYN 半开扫描（-sS） |
| 端口范围 | 指定 HTTP 常见端口列表（约 30 个） |
| 服务识别 | 不做 |
| OS 识别 | 不做 |
| 脚本扫描 | 不做 |
| DNS 解析 | 跳过反向解析 |
| 速率 | 中高速，配合超时控制 |

---

## 建议端口列表

```
80,443,8080,8443,8000,8888,9090,3000,5000,8001,8002,8008,8081,8082,9000,9443,4443,7443,7080,8180,8880,9080,10080,10443,3128,8090,8099,8800,18080
```

约 30 个端口，覆盖主流 Web 服务、反代、中间件管理端口。

---

## 关键参数策略说明

| 参数 | 作用 | 建议值 | 理由 |
|------|------|--------|------|
| -sS | SYN 扫描 | — | 速度快，不完成三次握手，相对低调 |
| -n | 跳过 DNS 反向解析 | — | 几千 IP 的 DNS 查询会额外浪费数分钟 |
| -T4 | timing 模板 | T4 | 激进但不极端，适合分钟级需求 |
| --min-rate | 最低发包速率 | 3000-5000 | 保证整体速度不被个别慢主机拖住 |
| --max-retries | 最大重试次数 | 1 | 默认值为 2，降低可节省时间，轻微增加漏报 |
| --host-timeout | 单主机超时 | 30s | 避免个别目标（filtered 多）拖垮整体进度 |
| --open | 只显示开放端口 | — | 输出更干净，减少噪音 |
| -oG 或 -oX | 输出格式 | 二选一 | -oG 适合 grep+awk 快速提取；-oX 适合程序解析 |
| -iL | IP 列表输入 | 文件路径 | 一行一个 IP，避免命令行过长 |

---

## 时间预估

| IP 数量 | 端口数 | 探测点总数 | min-rate | 预计耗时 |
|---------|--------|-----------|----------|----------|
| 3000 | 30 | 90000 | 3000 pps | 2-5 分钟 |
| 5000 | 30 | 150000 | 3000 pps | 5-8 分钟 |
| 5000 | 30 | 150000 | 5000 pps | 3-5 分钟 |

---

## 注意事项

1. **权限要求**：SYN 扫描需要 root 或 sudo 权限
2. **带宽占用**：min-rate 3000 约占用 1Mbps 带宽，对办公网络基本无影响；如需更保守可降至 1000
3. **输入格式**：IP 列表文件中一行一个 IP，不要带端口或协议前缀
4. **输出建议**：推荐 XML 格式，便于后续脚本提取「IP + 开放端口」对，拼接 URI 进行批量访问
5. **封禁风险**：部分目标可能有 IDS 或 IPS，存在被封或被记录的可能，从公司 IP 发起需自行评估合规风险

---

## 命令结构参考（参数组合示意）

```
nmap -sS -n -T4 --min-rate 3000 --max-retries 1 --host-timeout 30s --open -p <端口列表> -iL <IP文件> -oX <输出文件>
```

> 以上为参数组合结构示意，具体值根据实际情况填入。
